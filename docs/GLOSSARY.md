# Goblin Glossary

This document defines every key term, acronym, named algorithm, and library used in the Goblin algorithmic forex research platform. Its purpose is to make the project accessible to external contributors and reviewers who may not know the domain.

Terms are organized by category, then alphabetically within each category.

---

## Acronyms

| Term | Full Form | Definition |
|------|-----------|------------|
| **ADR** | Architecture Decision Record | A governing artifact that records a significant design choice, its context, the alternatives considered, and the rationale for the decision made. Goblin uses ADRs to document structural changes to the kernel. |
| **DSR** | Deflated Sharpe Ratio | A corrected Sharpe Ratio that adjusts for multiple testing, non-normality of returns, and length of track record. Prevents overstating statistical significance when many strategies are evaluated. Introduced by Bailey & Lopez de Prado (2012). A DSR ≥ 0.0 is required for promotion. |
| **EA** | Expert Advisor | A compiled trading robot deployed on the MetaTrader 5 (MT5) platform. Executes trades based on pre-defined rules in the broker environment. **Never an optimization algorithm.** In Goblin, EAs are generated from governed `CandidateDraft` specs and used for practice/deployment validation only. |
| **EvA** | Evolutionary Algorithm | Any population-based, bio-inspired optimizer used in Goblin's ML pipeline: CMA-ES, DE, GP, NSGA-II, NEAT. EvAs optimize parameters, discover rules, and evolve model architectures. **Never an MT5 Expert Advisor.** |
| **GMM** | Gaussian Mixture Model | A probabilistic model that assumes data is generated by a mixture of Gaussian distributions. Used in Goblin as the regime classifier (`ml/regime.py`) to label each bar as belonging to a market condition (e.g., steady, volatile, transitional). Introduced in ML-P1. |
| **GP** | Genetic Programming | A form of evolutionary computation that evolves expression trees (programs) rather than fixed-length vectors. In Goblin (ML-P1.5), GP evolves entry/exit rule trees from feature primitives using the DEAP library. Rules are evaluated on OOS profit factor; parsimony pressure limits tree complexity. |
| **HITL** | Human-in-the-Loop | A governance tier requiring explicit owner approval before implementation may proceed. Applies to ML-P2 and all ML-P3 sub-phases. Autopilot must deliver a full Go/No-Go Analysis Package and pass a self-audit before presenting it to the owner. |
| **MOC** | Map of Contents | An index note in Obsidian that links to related notes in a vault. Used in the Prophet D&D campaign management project in this repository's sister workspace. Listed here because the term appears in shared governance documents. |
| **OOS** | Out-of-Sample | Data that was not seen by a model during training. OOS evaluation provides honest performance estimates. In Goblin, OOS data comes from the held-out fold of purged walk-forward cross-validation — it is never used to tune parameters. |
| **PBO** | Probability of Backtest Overfitting | A metric derived from Combinatorially Symmetric Cross-Validation (CSCV) that estimates the probability that a selected strategy is overfit to its backtest data. Introduced by Bailey, Borwein, Lopez de Prado & Zhu (2014). Goblin requires PBO ≤ 0.35 for any model promotion. |
| **PF** | Profit Factor | Gross profit divided by gross loss over a backtest window. PF > 1.0 means the strategy is profitable. The primary fitness metric for Goblin strategy candidates. A PF ≥ 1.0 is the baseline promotional requirement; higher thresholds apply to parity-eligible candidates. |
| **SHAP** | SHapley Additive exPlanations | A unified framework for explaining machine learning model predictions, rooted in cooperative game theory. Assigns each input feature a contribution value for a specific prediction. In Goblin, TreeExplainer SHAP values are generated for every XGBoost signal filter model to verify that the model's decisions are attributable to real market features. |
| **SL** | Stop Loss | A pre-set price level at which a losing trade is automatically closed to limit further loss. Expressed in pips in Goblin. SL bounds are governed by `config/risk_policy.toml` and optimized by the CMA-ES optimizer. |
| **TP** | Take Profit | A pre-set price level at which a winning trade is automatically closed to lock in gains. Expressed in pips in Goblin. TP bounds are governed by `config/risk_policy.toml` and optimized by the CMA-ES optimizer. |
| **WFV** | Walk-Forward Validation | A time-series cross-validation method that respects temporal order: the model is trained on a window of past data and validated on the immediately following window. Goblin uses purged WFV with an embargo buffer to prevent look-ahead contamination and data leakage across folds. |
| **BH-FDR** | Benjamini–Hochberg False Discovery Rate | A multiple-testing correction that controls the expected proportion of false positives ("discoveries") among the set of rejected null hypotheses, at a chosen level `q`. Used in ML-1.6b to evaluate per-feature contributions of sequential features without inflating false-positive rate across the 6 candidate features. Less conservative than Bonferroni (which controls family-wise error rate). |
| **MDE** | Minimum Detectable Effect | The smallest true effect size a study is powered to reject the null at, given the noise floor and sample size. In ML-1.6.0, the Variance Pilot established `σ_PF=0.0083` and the MDE for downstream phases was set equal to `σ_PF` so that no claim of "improvement" can be made unless it exceeds the measured per-candidate noise floor. |
| **PSI** | Population Stability Index | A measure of distributional drift between two samples (typically training vs production), computed as the symmetric Kullback–Leibler divergence over a binned distribution. Values < 0.1 indicate stable, 0.1–0.25 indicate mild drift, > 0.25 indicate significant drift. Used in Goblin as a pre-deployment data-drift gate for model features. |

## Named Algorithms and Methods

| Term | Definition |
|------|------------|
| **CMA-ES** | Covariance Matrix Adaptation Evolution Strategy. A population-based, derivative-free optimizer for continuous parameter spaces. Adapts its covariance matrix to learn the shape of the fitness landscape, making it efficient for non-convex problems with dependencies between parameters. Used in Goblin (ML-P1) to optimize `stop_loss_pips`, `take_profit_pips`, `signal_threshold`, and `holding_bars`. |
| **CSCV** | Combinatorially Symmetric Cross-Validation. The statistical method underlying PBO calculation. Partitions backtest data into subsets and evaluates whether the selected strategy performs above median in the held-out partition. See Bailey et al. (2014). |
| **NEAT** | NeuroEvolution of Augmenting Topologies. An evolutionary algorithm that simultaneously evolves the topology and weights of a neural network, starting from minimal structure and adding complexity only when it improves fitness. Planned for ML-P3c (HITL-gated). |
| **NSGA-II** | Non-dominated Sorting Genetic Algorithm II. A multi-objective evolutionary optimizer that maintains a Pareto-optimal front of solutions. Used when optimizing for multiple objectives simultaneously (e.g., profit factor and drawdown). |
| **Purged Walk-Forward CV** | Walk-Forward Validation with an embargo buffer that removes training samples temporally adjacent to the validation window. The embargo prevents the model from learning patterns that only exist because they are close in time to the target — a common source of look-ahead leakage in financial ML. Goblin's embargo is `max(holding_bars, 10)` bars. |
| **White's Reality Check** | A statistical test that controls for data-snooping bias when multiple strategies are evaluated. Tests the null hypothesis that the best strategy has no predictive power above a benchmark. Goblin requires p ≤ 0.10. |

## Named Systems and Platforms

| Term | Definition |
|------|------------|
| **OANDA** | The canonical research data source for all Goblin alpha claims. Provides real bid/ask price pairs, real market spreads, and tick-count volume per bar. All training data is sourced from OANDA. MT5 data validates deployment readiness but never overrides OANDA-sourced research truth. |
| **MT5 / MetaTrader 5** | The broker-side deployment platform for Goblin's Expert Advisors. MT5 uses a single price stream plus a configured spread, making its data structurally different from OANDA's real bid/ask data. MT5 is used for practice validation and broker reconciliation only — never for research or alpha claims. |
| **Goblin** | The umbrella program identity for this project. The control plane lives in the `/Goblin/` directory. The deterministic Python kernel lives in `src/agentic_forex/`. The bridge namespace `src/goblin/` aliases to `agentic_forex` for convenience. |
| **`AF-CAND-0263`** | The locked overlap benchmark candidate. Its parameters are immutable. It serves as the reference point for comparing new candidate strategies in the overlap trading session. No ML/EvA work may modify it. |

## Libraries

| Library | Definition |
|---------|------------|
| **CMA (Python library)** | Python implementation of the CMA-ES optimizer (`pip install cma`). Used in `src/agentic_forex/ml/optimizer.py` for parameter optimization in ML-P1. Wraps the canonical CMA-ES algorithm by Nikolaus Hansen. |
| **DEAP** | Distributed Evolutionary Algorithms in Python. Provides the building blocks for genetic algorithms, genetic programming, evolution strategies, and particle swarm optimization. Used in `src/agentic_forex/ml/gp_rules.py` and `ml/primitives.py` for GP rule discovery in ML-P1.5. |
| **NumPy** | Numerical array computing library for Python. Provides N-dimensional arrays and vectorized mathematical operations. Used throughout Goblin for feature computation, matrix operations, and numerical results. |
| **pandas** | Data manipulation library for tabular and time-series data in Python. Provides the `DataFrame` and `Series` structures used throughout Goblin for OHLC price data, feature matrices, and backtest results. |
| **Pydantic** | Data validation and settings management library for Python. All Goblin configuration models, governance models, and API contracts are defined as Pydantic models. Provides runtime type checking and TOML-to-Python config binding. |
| **scikit-learn** | General-purpose machine learning library for Python. Provides base classifiers, `GaussianMixture` (used for GMM regime classification), cross-validation utilities, permutation importance, and the `LogisticRegression` baseline signal filter. |
| **SHAP (Python library)** | Python library implementing SHAP (SHapley Additive exPlanations) values. Uses `TreeExplainer` for XGBoost models to produce per-feature importance scores efficiently. Used in `src/agentic_forex/ml/train.py` from ML-P1 onward. |
| **XGBoost** | eXtreme Gradient Boosting. A highly optimized gradient-boosted decision tree library. Used in Goblin as the primary signal filter from ML-P1 onward, replacing the earlier RandomForest. Configured with `max_leaves=500` and trained with purged walk-forward CV. |

## Domain Terms

| Term | Definition |
|------|------------|
| **Backtest Overfitting** | The condition where a strategy appears profitable on historical data because it has been tuned — explicitly or implicitly — to the specific noise patterns in that data. A strategy that is overfit has no true predictive power and will fail on unseen data. Goblin's full overfitting defense stack (PBO, White's test, DSR, label randomization, adversarial validation) is designed to detect and reject overfit candidates. |
| **Candidate Draft (`CandidateDraft`)** | A Goblin governance artifact representing a proposed trading strategy. Contains all parameters (SL, TP, allowed hours, signal logic, etc.) in a governed schema. Must pass the full promotion pipeline before becoming a deployed EA. |
| **Embargo Buffer** | The number of bars stripped from the boundary between training and validation folds in purged walk-forward CV. Prevents leakage from temporally correlated samples. Goblin's embargo = `max(holding_bars, 10)` bars. |
| **Feature Extractor** | In the context of ML-P2 and beyond, a neural network whose output is an embedding vector fed into another model (XGBoost) rather than used as a standalone predictor. Goblin's temporal models (LSTM/CNN) are feature extractors only — they never make trade decisions directly. |
| **Latent Features** | The compressed numeric representations learned by a neural network's intermediate layers. In Goblin's autoencoder plans (ML-P3d), latent features encode patterns in raw bar windows that may not be captured by hand-crafted features. |
| **Overlap Session** | The London/New York trading session overlap (approximately 13:00–17:00 UTC), which historically shows elevated liquidity and volatility in major forex pairs. `AF-CAND-0263` was developed and optimized specifically for this window. |
| **Parsimony Pressure** | A penalty applied to genetic programming individuals that grow excessively complex. In Goblin's GP engine, parsimony is enforced via a fitness penalty per extra node and a hard maximum tree depth. Without it, GP evolves deep, uninterpretable trees that overfit. |
| **Parity** | In Goblin, parity means a strategy candidate has been validated to produce equivalent results in both the OANDA research environment and the MT5 practice environment, within governed tolerances (±0.30 pip price difference, ≥80% structural match). Parity is a prerequisite for live-demo deployment. |
| **Pip** | The smallest standard price increment in forex. For most pairs (e.g., EUR/USD), 1 pip = 0.0001 price units. Stop loss and take profit distances are expressed in pips throughout Goblin. |
| **Decision Log** | The append-only JSON-Lines record of every governed ML phase decision (`Goblin/decisions/ml_decisions.jsonl`). Each entry captures the decision id, phase, decision type (pre-registration vs completion), verdict, decided-by, decided-at, rationale, and an 8-field bias self-audit (`confirmation_bias`, `cherry_picking`, `anchoring`, `complexity_creep`, `sunk_cost`, `recency`, `narrative`, `survivorship` — each with `_considered:true` and a non-empty `_note`). Validated by `tools/verify_decision_log_schema.py` and protected by the `decision-log-append-only` CI job. |
| **Effect-size floor** | The minimum absolute effect size that a downstream ML phase must exceed to claim a true improvement. Locked in ML-1.6.0 to `0.0083` PF — the empirically measured per-candidate σ_PF across 10 seeds × 11 candidates. Any reported lift below this floor is considered indistinguishable from training noise and cannot be used to justify additional architectural complexity. |
| **σ_PF (sigma-PF)** | The standard deviation of profit factor across seeds for a single candidate strategy at fixed hyperparameters. Measured directly in the Variance Pilot (ML-1.6.0). Used as both the effect-size floor and the MDE for downstream ML phases. The locked value is `0.0083`. |
| **Sealed Holdout** | A chronologically-latest 20% slice of the canonical research dataset that is encrypted (Fernet AES-128 + HMAC) at the start of an ML phase block. The Fernet key lives **outside the repository** (under `~/.goblin/holdout_keys/`); only the ciphertext lives in the repo (`Goblin/holdout/ml_p2_holdout.parquet.enc`). The seal can only be broken by the ML-2.0 HITL re-gate ceremony. Per the autonomy policy, seal access is logged and limited; CI jobs (`holdout-access-audit`, `holdout-key-leak-check`) enforce that no key file is ever committed and no kernel code can decrypt the payload. |
